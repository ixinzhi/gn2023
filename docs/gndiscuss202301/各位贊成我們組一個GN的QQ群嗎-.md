# 1

.
為了調查本論壇是否有需要開設一個專用QQ群，故舉辦這次投票。
希望可以得出足供參考的結果。
因此，非常希望看到這主題的人都能夠來發表意見，至少也要投下你的一票。

有些人或許已經知道GN存在一個掛名的QQ群，但實際上這個群的性質更偏向私人社團。
因此這個投票的目的是詢問各位GN論壇是否需要開設一個「**全面開放且無限制資格**」的QQ群。

所謂QQ群，就是大陸騰訊公司所建立的一個雲端社群平台，參與者需要在電腦或手持設備上安裝QQ軟體。（或使用瀏覽器的web QQ）
對大陸地區的同好來說也許不陌生了，但其餘地方的朋友可能甚至沒聽說過QQ這軟體。
簡單來說，QQ的性質就類似MSN或skype，不過功能上還是有差距，使用的體驗上也有很大不同。
單純以一對一的即時通訊功能來說，我自己的感覺是QQ軟體沒有多大的特殊之處，但我必須先說明我並非深度使用者。
然而以多對多的層面而言，QQ群在我看來是相當便利的多人即時通訊平台。
可以對群內的所有在線者發送訊息、圖片，此外也有有趣的擷取螢幕畫面的功能，作為相同主題的清談場所應該是相當不錯。
論壇無法允許太水的留言，也無法允許過度離題或毫無營養的主題，因為論壇本身需要耗用資源去維護這些資料。
但QQ群不需要群主去維護這些資料，所有留言紀錄都存在各位自己的電腦裡、或者騰訊的雲端平台，因此氣氛上可以相當輕鬆。
以上是我個人之所以推薦QQ群的一點理由。
如果有人覺得還有更好的同性質平台或軟體，也可以推薦給我使用。

總而言之，這次投票的目的，就是調查有多少人願意成為這個論壇QQ群的成員。
而投票結果，除了決定是否開這個QQ群以外，也會決定本群使用何種等級的資源。
QQ群分成100人上限、200人上限與500人上限，我們將視此投票結果來決定開哪一種群。
人數多的大群當然是比較難建立的，所以我們必須有充足的理由去費這個勁。

選項一共七個，算是依照我的習慣分得較細了一些，不過最終依然會是看願意加入的人數有多少來決定。
在這個議題上我應該沒什麼理由去考慮投「不加入」的人們的意見。

順便說，依照目前我的QQ群使用經驗，除了交談以外，很多時候我們也會在QQ群作資源分享。
以目前論壇越來越難進行這類資源分享的現實來看，（FBI在瞪著你）
重點資源的分享日後全部移轉到QQ群上，應該是很有可能的事。
我個人是希望沒安裝QQ或對QQ有意見的人都可以多考慮一下。

本次投票將隨著「資源分享暫存區」在月底的結束同步停止。 <title>2</title> <link href="../Styles/Style.css" type="text/css" rel="stylesheet">

MSN的M群其實到今日都還是存在的，可是……大概已經好幾個月沒有人說話了吧？
這幾個月我收到發自M群的訊息全部都是廣告文＝ ＝

我不敢說誰的功能比較棒，但M群的使用體驗不如QQ群，在我看來是這樣。
從管理者的角度來看，M群還更難用。
所以應該不會再使用M群當主力的聊天場所…

GN官方M群的帳號是 [group27786@xiaoi.com](mailto:group27786@xiaoi.com)
可以自由加入與退出，但真的沒人在說話了……

話題回到QQ群上。
我不認為會有4000人選500這種問題，首先我們全部的活動會員就沒有4000人。
後台介面都看得很清楚，許多的帳號是各位在嘗試註冊時不小心多出來的。
光是這樣（有填寫註冊單卻沒有去e-mail認證）的帳號至少就有1000個。（不誇張就是這麼多）

再來，QQ群應該會謝絕語言不通的人加入。
坦白點講，這個QQ群就是要負責扛起一些論壇上「不能作」的事。
所以成員全部都挑選自己人是理所當然的。

太久不發言就踢，這點應該也會去執行。
所得與付出總不能是單方面的，連陪大家聊天都不肯真的也沒什麼好說的。

至於QQ群是不是會卡到論壇、影響論壇的人氣？
我想這在所難免，但我仍然得這樣做的理由上面也講了。
客觀來看論壇日後將越來越難以允許交流資源，同時水文與板聊也不可能允許存在。
我現在花在主機上的支出已經是GN創始時的四點五倍了，因為我們無法再用低階規格來營運。
（分水嶺大概從2010年開始，從那時起人多到無法再使用既有規格，才開始了一次次的升級與陣痛）
只有頂級的商業規格才能滿足我們這個論壇的消耗資源。
在這種情況下，我希望論壇的內容至少都是有一定水準的，或至少得有存在的意義。
也許開了QQ群，反而能讓已經相當緊迫的論壇資源喘口氣呢。

不過目前的投票人數還是偏少，希望大家多多出來投票。
我不會開什麼1群2群，假如投票結果出來顯示的是只需要開100人群，那就真的只是100人囉！
所以請各位多多支持一下吧！都點進來看了就投個票吧，感謝。 <title>12</title> <link href="../Styles/Style.css" type="text/css" rel="stylesheet">
