众所周知，百度盘资源被和谐是个很令人头疼的问题，不知道哪天下载的时候文件下面就会出现一句“该资源涉嫌违规无法下载”。更让人头疼的是，一旦被和谐，在此之前所有保存的人都无法下载。现在就来总结一下资源被和谐的几种可能………

应该如果不在线解压的话，百度是没有任何方法和谐一个加密的压缩文件的。因为破解一个密码非常的消耗时间资源，所以除非有人提供这个密码，百度是看不到的。

而举报，应该不算太大的问题吧……？我具体不太了解，百度网盘里举报内容会有奖励么？所以有人会下载完成后举报么？还是只是为了增加资源的稀缺性？

还有，存两份百度盘，如果两份的密码一样的话，应该是没有用的。因为百度一旦和谐了一个加密文件以后，很容易就可以在自己的系统里找到一样的文件，然后和谐掉这些。

最后的最后，百度作为服务商，如果想要做的狠，还有最终手段。那就是标记用户。就是说，像你和我这些经常下载和上传敏感内容的用户都会被自动标记上。一旦有足够数量的敏感人员与一个文件互动，那么就可以自动默认这个文件是个敏感文件了。当然，我知道的会这么做的服务只有Youtube。而且我觉得百度应该不太可能做的这么狠。虽然百度因为上层压力会需要把过于明显的色情内容处理掉，但是毕竟我们这些处理黄色内容的人，都是百度最重要的用户（因为用下体思考的时候一般都比较人傻钱多），所以百度本身应该是非常不希望我们离开百度，用别的方法存储资源的。

整体来说，最好的方法还是多有一些“人”作为节点。就是说，鼓励大家多分享分流转存资源。把百度盘上发过一遍的资源在腾讯盘上再发一遍，在Mega上再发一遍，在Exhentai上再发一遍，以此类推。

如果是发布者个人可以做到的事情的话，应该是更改后缀吧？就是说，发的内容把后缀直接删掉，让下载者自己在后面加上“.zip”。或者也可以加上一些假的没有意义的后缀，比如“.gts”什么的。这样应该相对非常保险，而且百度盘应该没法在线解压无后缀或后缀不正确的文件吧？缺点就是可能这会让手机用户，或什么奇怪的操作系统（如谷歌本）的用户很难受。 <title>3</title> <link href="../Styles/Style.css" type="text/css" rel="stylesheet">

# 4

这个我稍微研究过，感觉有点发言权，我也来总结一下：
首先百度没什么理由直接封禁一个带密码的压缩包，因为没法直接从压缩包看到内容，而要解压带密码的压缩包必须得暴力破解，密码一旦复杂就会非常消耗资源，所以得不偿失。因此一个带密码的压缩包被和谐了肯定是因为有人在线解压了。
如果一个人坚持要在线解压或者想举报的话你是阻止不了他的，双层密码压缩也只能增加被封难度，不是百分百安全，而且对于拿资源的人还很麻烦。
7z后缀的没法在线解压吗？没听说过，谁去给咱验证一下？
另外将后缀名改成.jpg或.mp4等无关格式的方法也是没用的，因为文件的格式信息是被写入文件本身了的，只需要一点小技巧就可以看到，然后将后缀名改回来就能看到内容了。
不过将压缩包命名为其他无关名字确实是有效的。不知道你们知不知道一个叫扶她奶茶的人，他可是本子界的大佬，一生（人还活着）分享了数不胜数的本子，而且资源绝大部分都在盘上躺的好好的。为什么？因为他的所有资源都被命名为“Windows修复补丁（后接本子发售日期）”，这样就算有人举报了，如果审查的不严格，看一眼标题发现与违禁不沾边或许就不管了。但是如果压缩包的名字很直白的就是“苍井空”，或者一堆日文，那很容易引起审查员的警惕，就会去看一看内容，然后资源就没了。
因此我有个建议，压缩完成后将压缩包的名字改为与内容无关的非违禁文本，而且最好是让人一看就明白是什么东西的，最好可以跟学习沾上边，比如“八年级下语文考试复习题”，“计算机二级考试模拟题”什么的，这样可以大大增加隐蔽力度。 <title>5</title> <link href="../Styles/Style.css" type="text/css" rel="stylesheet">
