# 1

我是那個在每個Po遊戲的人下面長篇大論如何做遊戲自己卻只推出個諷刺糞作的傢伙

我一直以來都想逼自己生個作品，但是想法太多太雜反而什麼都沒做出來，
所以今天決定直接來開許願池，看看大家想要什麼
以前也不是沒想過這樣發文徵求靈感和需求，
但我總怕留言會是各種「我想要可以互動的陰毛」這種毫無方向性的屁話，
或是「我想要2D-HD重製縮小學園」這樣把別人當哆啦A夢的幹話

至於為什麼今天突然發文了，是因為我突然發現自己最能有貢獻的地方
論壇上絕大多數遊戲開發者都是使用不同版本的RPGMaker，還有一些3D開發者，
幾乎沒有人用汎用遊戲引擎（General Purpose Game Engine）在做2D遊戲的，
所以今天就在這裡問問看：
**各位有沒有什麼想玩的2D遊戲、想要的體驗，**
**是RPGMaker做不出來的？**

其實技術上不能說「做不出來」，
但就像鉛筆技術上不是不能當輪子一樣，你得非常閒或非常絕望才會考慮把這當工具
類魂遊戲、彈幕遊戲、射擊遊戲、平台遊戲、格鬥遊戲……
這些是壇友們用RPGMaker大概不會做，大概也不該碰的領域，卻在我的技術範圍內

非格狀運動、投射物、物件加速度、粒子特效……
這些東西我都看得太理所當然了，但說不定其中就有某個壇友夢寐以求的效果在裡面
所以以下開放委託，當然不保證實現，但反正許願不用錢
沒人回我就繼續過日子，有太多可行方案就開票池，總之請踴躍留言

你這不是願望，只是趁亂發問，但好……
首先，你的「2D沒辦法做探索感」的前提本身就有問題
你舉例的曠野之息，一開始就是用1986年的初代薩爾達傳說當範本的，
那可是紅白機的遊戲，當初設計來就是要重現宮本茂小時候探索山洞時的感受
然後你玩縮學的迷茫感，那不是種探索感嗎？
如果縮學缺的是標誌，為何必須是3D的標誌才能達到你要的效果？
我不懂你想問啥，所以很難回答……

然後皮膚探索……因為一般人想像的皮膚是平的，所以做出來也是平的
用個現實點的例子講，如果你做個「機場跑道探險遊戲」，只看得到一大片水泥平地，也會很無聊、很空虛，
所以要怎麼讓人感覺到「探索」，就是要有相應的障礙與挑戰
比方說，如果你想要強調崎嶇不平，那就讓主角拖個箱子前進，或是弄個走上坡會耗體力走下坡會滑的設定，保證崎嶇感馬上湧上心頭

後面的問題，
1.當然可以，你只要把場景設好，然後丟一無所知的玩家進去就好
開發者經常擔心玩家迷失方向，塞一堆提示、引導、內心獨白確保有跟上狀況，
但如果你的場景搭建一開始就是為了探索而設計的話，玩家就該可以遵照自己的直覺挖掘出遊戲的運作方式
那邊那個箱子有什麼？那邊那個NPC跑去哪了？這桶水倒到剛才的火裡會發生什麼事？
你就限制自己一個字都不能打、一個箭頭都不能放，試著單純用場景佈置來引誘玩家行為，再想要怎麼保持玩家興趣
就想像玩家是鴿子，你要想辦法在不嚇跑玩家的前提下，保持距離一直丟土司，引誘到你想要的方向上……
……喔對，你問的是RPGMaker，去玩幾個經典恐怖遊戲應該就會有概念了

2.超微小不是我的守備範圍，但我盡量……
如果你只是把玩家丟到一片超巨大的皮膚上，體驗上跟丟到沙漠正中央沒有兩樣；
就算上面有什麼皮屑或微生物，只要玩家沒辦法在大腦中認知到這是超迷你的身體殘渣，那些物件也只會是視覺雜訊
所以，優先讓玩家認識到「雖然幾乎認不出來，但是這裡是在人體上」，應該對體驗會有不少正面影響
比方說，先有個正常尺寸和女角色在實驗室見面的故事，
然後對方跑去漏肚子睡覺，但是實驗器材出問題，所以玩家跑去搖女主角，
然後就縮小掉到肚子上了，然後階段性地變得越來越小，然後途中還打翻某種東西到肚子上，
所以當主角被縮到很小，並且在周圍看到了自己剛才打翻的東西，所以可以認知到自己在皮膚上……
……之類的，我不知道你的點在哪，所以只能給沒方向的建議……

……你看看你害我又回頭講理論了

以我目前玩到的RPG maker 遊戲來說，

1.遊戲必須是作者自己喜歡的：
(一方面是動力，支持自己做下去、另一方面是自己都不喜歡的作品，沒有能力說服其他玩家喜歡。)

2.把想要呈現的東西表達並深化：
(也就是去除不必要的內容，例如：為人詬病的練技，跑地圖。除非想要表達的是不斷的失敗挫折，或是挑戰強敵來獲得成長等，含意)

3.檢視自己的作品最好的方法是，「我能不能定期重玩回味?」：
(有些故事吸引人，或是遊戲性獨特，或是單純很戳自己的性癖)、
(這一方面可以認可自己做的作品，二方面也有機會繼續發展DLC，或是檢討自己的不足之處，讓下個作品能嘗試補足這個缺陷。)

> [琉璃 發表於 2022-12-5 14:51](https://giantessnight.cf/gnforum2012/forum.php?mod=redirect&goto=findpost&pid=521120&ptid=34510)
> 今天我拿unity尝试了下，发现从unity开始从0做真的很难。才解决主角行走问题。还有地图编辑的问题，因为打 ...

如果沒碰過Unity但想學，推薦這個前幾天才出的極基礎教學： [https://youtu.be/XtQMytORBmM](https://youtu.be/XtQMytORBmM)

裡面也提到了學習遊戲引擎該做的事，就是學習自己想要做出遊戲的話「無論如何都得學會的功能」，
例如如何聽取玩家輸入、如何生成與消滅物件、如何讓兩個東西碰撞、如何指定播放音效時機……

我先列出你想的效果需要的邏輯
1.無縫地圖
這個最不用擔心，就把地圖做大就好，地圖轉換反而比較難做
除非你的地圖真的真的很大，大到會讓記憶體（內存）或處理效能不夠，
那就要有讓離玩家遠的地圖自動解除功能，等玩家靠近再偷偷讀取的功能，
但這種事遇到的時候再想就好，不用提前擔心最佳化（優化）問題
2.鏡頭移動
把鏡頭綁在玩家身上就好，玩家走到哪就拍到哪
如果想要順暢的鏡頭移動，就去研究一下lerp（插值），
這解釋起來比較麻煩，查不到的話再說
3.對話框
……用Plug-in就好，這東西自己做很複雜也很麻煩，
除非你想要什麼超級特殊的效果（像是對話框裡的文字會飛出來打人）不然自己做真的不值得
4.觸發點
放一個區域（類似感應器的東西），設定當玩家進入時會發生事件
這個區域通常是玩家看不見的，所以視覺上可以用別的方法呈現
5.碰撞
就Collider Component，隨便跟個教學應該都會講……
……你該不會是沒跟教學直接硬幹吧？
6.潛行系統
這就超出引擎範疇了，這是遊戲的邏輯設計
單講要怎麼判定玩家有沒有被發現很簡單，就用會移動的區域判定玩家有沒有重疊就好，
但是被發現以後AI要怎麼行動、被發現以後要不要有回歸平靜的方法、
在發現與平靜當中要不要有其他階段、其他NPC要不要同步行動……

你也根本不用怀疑，事实上，我就是没有开发过游戏。
我也从来没有以创作者的角度讨论游戏，我始终围绕的是“我作为玩家视角看到的游戏”在讲，一个“能满足我最低要求（即“玩得下去”）的游戏”至少该有个什么样子。我对引擎的认知全部来自于我为了解决遇到的问题自己边搜边琢磨的那点零散边角料，我指出“不同引擎可以达成相同效果”不过是为了说明“玩家往往并不在乎游戏是由哪个引擎制作”这一事实，而不在于这一操作事实上是否可取。一个引擎实现不了，就用另一个引擎实现，这很正常。强行在不太适宜的引擎下实现一些功能，只要能做到也不是不行。

选择合适的引擎当然是制作的一环，但就像你说的，我这“从来没开发过游戏的人”根本不清楚究竟godot，虚幻或是unity各自擅长什么，也不清楚哪种引擎制作哪类游戏会更加省时省事，我想绝大多数玩家都不会特别清楚。既然这是制作层面的事，何必要拿出来交由玩家讨论？外行提出一些不切实际的想法就让你感到不可接受的话，何必在这里开一个“许愿池”呢？

就像你说的，关键是有做还是没做，我至少知道每种引擎的工程文件是不一样的，也就是说，点下新建工程这个键的时候，就已经该确认要制作的游戏是在哪个引擎上实现比较妥当了，至少应当尝试过预期的模式实现是否有困难，否则这规划做了和没做有何区别？至少我玩过的和见过的“开发中途换引擎”的游戏十个有九个结局是暴毙，毕竟在决定换引擎的那一刻，前面一切进度约等于归零，在玩家视角里，一个尚且没影的游戏说所谓“开发的方向和最初出现了偏差所以要推倒重来”，和从一开始就没做有什么区别呢？游戏在它的测试放出来之前，对玩家来说就是压根没开始，等于零，等于没做。我前面也提过，黄油是“最简单难度”，不会有谁非得去挑剔它的细节如何如何，就是真的甩个被技术困难卡死的半吊子游戏出来然后说明情况弃坑，只要它目前展现的部分像个游戏，我一样买账。制作者对自己高标准严要求当然是好事，但高标准严要求到“真人永远不露相”的地步，说白了就跟mirror2的那档事一个毛病，做得太少想得太多。

你大可把我的回复当成我对你所述“许愿池”的要求，无论是什么引擎，什么类型的游戏，“首先这里得有个游戏，其次它得像个游戏”。我甚至不太敢对你提出再多的想法，毕竟这“许愿”的说法实在过于宽泛，而实际操作又过于局限（事实上我提过，比如RTS？），进怕无法实现，退怕拘束思路，倒不如啥也不说来得简单。

开发中遇到任何形式的困难，讲明白了实际上都能理解，毕竟确实不是容易的事。但是说到底，创作者给玩家讲长篇大论而又不给玩家放测试，最后往往就是对牛弹琴，无论怎样复杂的原因，说多了最后只会被归结为两个不太好听的词：“画饼”和“卖惨”。说直接点的话，对于圈内的游戏我从来都是以恳切的支持者自居，有想法就是好事，有困难也能理解，有需要可以提出，力所能及的都可以帮，但这出“许愿池”的操作就是负印象起手，因为我以玩家视角得出的判断是：你自己都不清楚你想做个什么游戏的时候，就已经在考虑要发挥哪些引擎的优势做出多高大上的作品来了，这是否与你自己“对开始来说重要的是有做还是没做”的说法背道而驰？做游戏只要打算公开那就是给人玩的，不是做给自己用来炫技赌气自嗨的，如果你先前真的有认真打算过做个游戏，又何必设置这种类似于立军令状或者按你的话说“找个理由”的赌气桥段？

说句不好听的，如果你真的有认真投入精力在做个自己的游戏这件事上，为何还能高强度浏览论坛并且写下大量大段长篇？没有人天天催进度的日子真轻松，对吧？
