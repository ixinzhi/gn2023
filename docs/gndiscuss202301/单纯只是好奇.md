# 2

自然是严谨一点好，严谨的设定才让人有代入感，但严谨不意味着繁琐，而是人物的行为和事情的发展都有合理的解释且逻辑自洽。 <title>3</title> <link href="../Styles/Style.css" type="text/css" rel="stylesheet">

# 3

跟你的寫作性質有關
如果你一直解釋現在發生的事 讓角色不停地對話討論行動策略 那世界觀和角色設定最好圓得起來
如果你整篇文只是從不明究裡的角色視角欣賞巨人自慰 那大概用不到多詳細的設定 <title>4</title> <link href="../Styles/Style.css" type="text/css" rel="stylesheet">

个人浅见：
要么没有设定(即gts的存在是无需说明/不言而喻的)
要么就尽量设计一个逻辑自洽的设定
后者在某些时候比单纯的福利更能让人爽(小声) <title>7</title> <link href="../Styles/Style.css" type="text/css" rel="stylesheet">
