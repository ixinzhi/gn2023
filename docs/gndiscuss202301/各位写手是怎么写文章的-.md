# 10

**人物——大纲——福利点**
最简单的流程就是这样了
想人物设定时经常就把故事背景这些麻烦事一起干了，效率高；
写个大纲让故事有个起码的开头、发展、结尾，但因为太监多发，没有结尾也很正常，然后为了爽单刀直入没有开头铺垫也很正常，你看写YY文多省事；
福利点，基本上是写给你自己的福利点，看的人刚好电波对上了也觉得爽更好，为了让美好的事情更频繁的发生，可以好好剖析一下自己为什么兴奋，把这些兴奋点都罗列出来，合理的塞到福利片段里并体现出来。

**举了例子**：气味控、长靴控、百倍内爱好者。
兴奋点：妹子的脚臭、妹子的原味长靴、被缩小到1-10厘米之间
福利片段：爬妹子的靴子，不小心跌落到靴内
突出：妹子靴子的巨大和小人的渺小、小人被妹子的脚臭折磨、小人因为脚臭反而兴奋起来

爬靴子的理由随便编编就好了，比如手机在椅子上，需要沿着妹子的靴子爬上去拿手机求救
怎么突出，如果真的有耐心的话，就靴子和小人的对比这点，光是爬靴子的过程就可以分为：走到靴子前、爬上鞋面、开始攀爬、登上靴沿、坠入靴内、靴内探索。中间可以加入小人克服种种困难的过程，就靠你想象力了，甚至可以插入回忆杀，这双靴子还是小人送给妹子的礼物，靴子上的污渍还是昨天和妹子一起去逛街粘上的云云；有能一点还可以让小人在爬的过程中越变越小，中途跌落以更小的姿态再爬一次也不是不可以。

我是分为直写法，聊天记录法，填充法三种
直写法，直接写，剧情连贯但是经常写着写着就崩了
聊天记录法，和别人对戏语c文游，然后把聊天记录变成文，非常麻烦繁琐，而且多个人的聊天记录要整合起来很难，再加上原创，总之很麻烦，但是作品最好
填充法，写个大纲，在大纲里加内容，这种效率最高 <title>12</title> <link href="../Styles/Style.css" type="text/css" rel="stylesheet">
