> 简单说一下，我个人认为，论制作成本来说：

論創作者／開發者成本來講，在有一定基礎品質的前提下，這個排序是有一點問題的。

以美工來講，單畫一張圖（約一～兩天工作天）大概有 5000 成本；如果是接畫冊之類的，能一口氣翻到近十萬。

漫畫以一集十幾張稿來講，大概兩、三萬元跑不掉，量多畫到一本的話翻到幾十萬都可能。

動畫泡麵番每集成本比較低，只有數萬元左右，質量好一點的可能會破十萬；如果是半小時的正常番，普通質量的一集大概近百萬跑不掉，原畫數多且做工細緻的破好幾百萬都有可能；若做成動畫電影，基本上都是千萬起跳的。

小說雖然只有純文字，不過寫成輕小說的量的成本少說也有幾十萬；寫出半千頁厚的（哈利波特、魔戒之類的）則是直接翻到百萬以上。

遊戲如果是用 RGB maker 做出來的簡易版（鮮少更動內部 code），10 分鐘極短篇成本約幾萬出頭；1~2 小時的短篇十萬多；5~6 小時的中篇差不多跟半小時的動畫一樣；12 小時以上的長篇是百萬起跳。
有去改寫內部模組或是動用大量美工的中、長篇（如帽子世界、東方年代記）直接百萬起跳；flash 遊戲報價跟 RPG maker 差不多，不過成本與 coding 能力的深度（直接影響遊戲可玩性）比較有關係。
以劇情為主的遊戲，在短篇部分的成本雖然會比以遊戲性為主的低一點，不過一旦長度拉長到中、長篇，很快就會超過以遊戲性為主的報價（技術的可複製性、創意的不可複製性）。

Unity 或是 3D 遊戲的成本大概是平面遊戲的四、五倍，像之前 Dreams 雖然是短篇，論成本來講幾十萬應該也跑不掉。如果是配上遊戲專用引擎的大型 RPG 長篇遊戲（大部分的線上遊戲），至少都千萬起跳、甚至到破億的；大陸很多看起來長的一模一樣的免洗筷式線上遊戲，其實成本也都有百萬以上（跟一般成本以千萬計價的線上遊戲，算便宜了）。

真要排序的話，大概是以下的關係（動畫跟中、長篇以上遊戲，以創作團隊角度來看）：

一張圖＜一集漫畫＜畫冊＝一本漫畫＝數分鐘泡麵動畫＝短篇遊戲＜一本輕小說＜中篇遊戲<半小時長度動畫＝長篇小說<長篇遊戲＝免洗筷線上遊戲<動畫電影＜線上遊戲＜營運 N 年的線上遊戲（再開發成本）
