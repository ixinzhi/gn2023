# 1

很多同好们都知道，我维护这一个免费的资源分享网盘。
前两天我为了进行数据同步，断开了网页的访问。但是我今天重开访问后查阅解析商数据发现，在过去的24小时内处理了来自708个独立IP的八万余条请求记录。
那么事情很显然，有一大帮人在用自动程序7*24小时对我的页面进行抓取。

我已经强调过，我所使用的微软文件存储服务风控十分严格，高强度的读写极易造成存储的暂时不可用。

我存储的文件中，为了方便在线观看，有大量已经解压的小文件（图片，文本），如果使用爬虫抓取文件，会导致高频率的风控和节流，造成的结果就是谁都没法用。

我完全可以使用高强度的反爬虫设计，但这样只能进一步降低用户体验并且浪费我自己的时间。 <title>2</title> <link href="../Styles/Style.css" type="text/css" rel="stylesheet">
