# 4

我也遇到这个问题了。
我自己觉得，把意思翻译出来是最重要的，即使可能会丢失一些风韵，首先得保证所有基本意思都能准确表达。
然后考虑的是呈现。不同文化的表现方式是不同的，如何有机转换也是需要考虑的一点。我认为有可能的话尽量找类似的表达，不行的话可以直译，甚至备注，有些精巧的表达必须是某种语言才有的梗，不下很大功夫无法转换，这时候考虑到时间成本，还是尽量找眼下最优解比较合适。
合格翻译不难，把意思翻译出来就行。要做好的翻译才难。
而更多的在实践翻译中摸索门路，会是更好的方法，毕竟实践出真知。 <title>5</title> <link href="../Styles/Style.css" type="text/css" rel="stylesheet">
